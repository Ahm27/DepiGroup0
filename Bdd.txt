-------------------------------------------
  # Login Module
  # -------------------------------------------

  Scenario: Successful login with valid credentials
    Given the user is on the Login page
    When the user enters a valid username and password
    And clicks the "Login" button
    Then the user should be redirected to the dashboard
    And a welcome message should appear

  Scenario: Unsuccessful login with invalid credentials
    Given the user is on the Login page
    When the user enters incorrect credentials
    And clicks the "Login" button
    Then an error "Invalid username or password" should appear

  Scenario: Login attempt with blank fields
    Given the user is on the Login page
    When the user clicks "Login" without entering credentials
    Then a message "Username and Password are required" should appear

  Scenario: Session timeout after inactivity
    Given the user is logged in
    When the user is inactive for more than 10 minutes
    Then the system should automatically log them out
    And display a session timeout message

  # -------------------------------------------
  # Registration Module
  # -------------------------------------------

  Scenario: Successful registration with valid details
    Given the user is on the Registration page
    When the user enters all required details correctly
    And clicks "Register"
    Then a confirmation message "Account created successfully" should appear

  Scenario: Registration with missing mandatory fields
    Given the user is on the Registration page
    When the user leaves required fields blank
    And clicks "Register"
    Then the system should display "Required fields missing"

  Scenario: Password and confirm password mismatch
    Given the user is on the Registration page
    When the user enters mismatched passwords
    And clicks "Register"
    Then the system should show "Passwords do not match"

  Scenario: Registration with existing username
    Given an existing username already exists in the system
    When the user tries to register with the same username
    Then an error "Username already exists" should appear

  Scenario: Invalid phone number or ZIP code format
    Given the user is entering registration details
    When invalid phone number or ZIP code format is entered
    Then the system should highlight invalid fields

  # -------------------------------------------
  # Forgot Login Information Module
  # -------------------------------------------

  Scenario: Successful recovery with valid data
    Given the user is on the "Forgot Login Info" page
    When the user enters valid First Name, Last Name, Address, City, State, ZIP, and SSN
    And clicks "Submit"
    Then the system should display the recovered username

  Scenario: Failed recovery with incorrect data
    Given the user is on the "Forgot Login Info" page
    When the user enters incorrect SSN
    Then the system should display "No record found"

  Scenario: Recovery with blank required fields
    Given the user is on the "Forgot Login Info" page
    When one or more fields are left empty
    Then the system should display "All fields are required"

  # -------------------------------------------
  # Account Services Module
  # -------------------------------------------

  ## --- Open New Account ---

  Scenario: Successfully open a new checking account
    Given the user is logged in
    And has at least one active account
    When the user selects "Open New Account"
    And chooses "Checking" as account type
    And selects a source account with balance ≥ $100
    Then the new account should be created successfully
    And a confirmation message should appear

  Scenario: Attempt to open account with insufficient funds
    Given the user has less than $100 in the source account
    When the user tries to open a new checking account
    Then the system should show "Insufficient balance"

  ## --- Transfer Funds ---

  Scenario: Successful funds transfer between accounts
    Given the user is logged in
    And has two active accounts
    When the user enters a valid transfer amount
    And selects different source and destination accounts
    And clicks "Transfer"
    Then the amount should be deducted from the source account
    And credited to the destination account
    And a success message should appear

  Scenario: Transfer with insufficient balance
    Given the user is logged in
    When the transfer amount exceeds the available balance
    Then the system should show "Insufficient funds"

  Scenario: Transfer with invalid or same account selected
    Given the user is on the transfer page
    When the user selects the same account as both source and destination
    Then the system should display "Invalid account selection"

  ## --- Bill Payment ---

  Scenario: Successful bill payment
    Given the user is logged in
    When the user enters valid payee details (name, address, account #, amount)
    And clicks "Send Payment"
    Then the payment should be processed successfully
    And a confirmation message should appear

  Scenario: Bill payment with invalid payee data
    Given the user is on the Bill Pay page
    When invalid or missing payee details are entered
    Then an error "Invalid payee information" should appear

  Scenario: Bill payment exceeding account balance
    Given the user’s account balance is low
    When the user tries to pay a bill greater than balance
    Then the system should reject the payment
    And show "Insufficient funds"

  ## --- Find Transactions ---

  Scenario: Find transaction by ID
    Given the user is logged in
    When the user enters a valid transaction ID
    And clicks "Find"
    Then the matching transaction should be displayed

  Scenario: Find transaction by date range
    Given the user is logged in
    When the user enters a valid start and end date
    Then all transactions within that range should be displayed

  Scenario: Find transaction by invalid amount
    Given the user is on the Find Transactions page
    When the user enters a non-numeric amount
    Then the system should show "Invalid amount"

  ## --- Update Contact Information ---

  Scenario: Successfully update profile details
    Given the user is logged in
    When the user updates contact details and clicks "Update"
    Then the profile should be updated successfully
    And a message "Profile updated" should appear

  Scenario: Attempt to update with invalid phone number
    Given the user is logged in
    When the user enters an invalid phone number
    Then the system should reject the update
    And display "Invalid phone number"

  ## --- Request Loan ---

  Scenario: Successfully request a loan
    Given the user is logged in
    When the user enters valid loan amount, down payment, and selects a funding account
    And clicks "Apply"
    Then the loan request should be submitted successfully
    And a confirmation message should appear

  Scenario: Loan request exceeding eligibility
    Given the user is logged in
    When the user enters a loan amount greater than eligibility limit
    Then the system should show "Loan request exceeds maximum allowed limit"

  # -------------------------------------------
  # Logout Module
  # -------------------------------------------

  Scenario: Successful logout
    Given the user is logged in
    When the user clicks "Log Out"
    Then the session should end
    And the user should be redirected to the login page

  Scenario: Logout and back navigation
    Given the user has logged out
    When the user clicks the browser back button
    Then the system should not allow access to previous pages
    And should redirect back to login
